How to read a micro-op

Each T-state below names one micro-op from the 57-entry dictionary. The name reads SRC→DST @mode — a single bus transfer — followed by any edge-actions taken on the closing (write) edge. A name with no → (e.g. ISET, SHIFT C) still occupies a T-state but only moves a pointer, shifts TOS, or writes flags.

Alu→Tos @Adr ALU=IR NZ C — drive the '181 result (function from the opcode) onto the bus, latch it into TOS, address the ADR register, and write N/Z/C on the edge.

ALU operation — carried by the opcode

Every ALU-quadrant opcode (0x40–0x7F) runs the same micro-op sequence, so the sequence never tells you which function it is — the '181 function rides the opcode bits, wired straight to the ALU: opcode = 0x40 | (M<<5) | (CN<<4) | S[3:0], i.e. 0 1 M CN S3 S2 S1 S0. M = 0 arithmetic / 1 logic, CN = carry-in, S = the four '181 select lines. N/Z are always written; C is written for arithmetic and held for logic. Every unnamed slot is a valid ALU #n realising that '181 function.

SHIFT operation — carried by the opcode

All five shifts decode to the single micro-op SHIFT C (idx 56). The variant is selected by raw IR bits: fill from IR[3:2] into the '153 fill mux, direction from IR[0] into the '194 and the carry tap. In-place on TOS; N/Z written, C from the bit shifted out.
