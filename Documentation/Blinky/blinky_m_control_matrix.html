Blinky-M — μROM Control-Line / Instruction Matrix

Superseded at rev 14. This matrix describes the retired EEPROM control store — 32 μROM lines against instructions. Rev 14 replaces that store with a two-stage GAL sequencer and a 57-entry micro-op dictionary; the control artifact is now the dictionary, documented in Blinky_M_GAL_Control.md §6. Notes

  • Purpose. Incremental build planning: pick the instruction set for a bring-up stage, OR its rows, and the marked columns are the hardware that must be alive. NOP needs only PC, its address drivers, the SRAM path, IR, and the sequencer; NIP adds nothing but the DSP counter (it touches no memory at all).
  • /TRST appears on every row by definition — kept for completeness.
  • COND and INTP are not control lines. They are μROM address bits; Bcc therefore shows the same line usage as JUMP, and the entry row is selected by INTP, not by a strobe.
  • ALU “op” marks that the '181 result or flags are consumed; the S/M/Cn lines are driven in every microword but are don't-care elsewhere.
  • HLT (ratified). The spare DST code is the HALT strobe — one wire to the clock module's halt flip-flop.
  • Shift C source (ratified). A second flags '157: one section taps the outgoing bit (Q7 vs Q0, steered by raw IR[0] — the opcode map encodes direction), one section selects the C input ('181 Cn+4 vs shift bit, C_SRC = TOSMODE-shift). Zero new μword lines; +1 package.
  • Shift N/Z (ratified). Shifts write C only; N/Z held. TST covers the rare case where flags of a shifted value are wanted.
  • Registered INTP (ratified). A0 of the μROM is a flop (spare interrupt '74), sampled at the fetch edge — the entry sequence cannot be torn when ISET clears the pending condition. B in the pushed flags byte = /INTP.
  • Flags hold-feedback (ratified). The '377's clock enable is common to all bits; per-flag masking (NZ_WE / C_WE) is realised as D = WE ? new : Q via a third flags '157. Core total 56.
  • PGSEL columns double as a memory-page usage map: which instructions touch frames (00), I/O (01), the data stack (02), and the return-stack lanes (03–06).
